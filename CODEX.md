# AI Works Lab ゲーム開発 Codex共有メモ

## 目的

このリポジトリは、AI Works Lab のゲーム開発プロジェクトです。

Python、Pygameを使った小さなゲーム開発から始め、将来的には他の言語やゲームエンジンも活用して、スマホ、パソコン、タブレットでも遊べるゲーム制作を目指します。

GitHub、X、note に開発記録を残しながら、学習・実績作り・発信・将来的な収益化につなげます。

目的は、ゲームを完成させることだけではなく、以下を積み上げることです。

・Python学習
・ゲーム開発の実績作り
・AI Works Lab の発信強化
・GitHubの開発履歴作成
・X、noteでの開発記録発信
・将来的な収益化の土台作り
・スマホ、パソコン、タブレットで遊べるゲーム制作への発展

## 開発方針

現在は、Python と Pygame を使って基礎を学びます。

ただし、このプロジェクトは Python だけに限定しません。

将来的には、目的に応じて以下のような技術も検討します。

・Python / Pygame
・JavaScript / HTML5
・Godot / GDScript
・Unity / C#
・その他、スマホ・PC・タブレット対応に向いた技術

ただし、Codexはユーザーが明確に依頼するまで、勝手に使用言語や開発環境を変更しないでください。

## 現在のゲーム

## Codexに守ってほしいこと

### 基本ルール

・現在の機能を壊さない
・勝手に大きな機能を追加しない
・1回の修正では、なるべく1つの目的に絞る

## ゲーム開発の優先順位

このプロジェクトは、いきなり完成度の高い作品を目指すのではなく、小さく改善していきます。

優先順位は以下です。

1. まず動くこと
2. 操作しやすいこと
3. 分かりやすいこと
4. 遊びやすいこと
5. 見た目を少しずつ良くすること
6. 発信しやすい形にすること
7. 将来的に他の端末でも遊べる形を考えること
8. 収益化につながる可能性を残すこと

## 技術選定ルール

Codexは、技術を選ぶ場合、必ず以下を説明してください。

・なぜその技術を使うのか
・スマホ、PC、タブレット対応に向いているか
・初心者でも続けやすいか
・現在のコードをどの程度活かせるか
・悪い影響や難しくなる点はあるか

ユーザーが明確に許可するまで、勝手に技術変更しないでください。

## 勝手に追加しないでほしいこと

以下は、ユーザーが明確に依頼するまで勝手に追加しないでください。

・大量の新キャラクター
・複雑なステージ制
・画像素材の前提追加

・オンライン機能

・スマホアプリ化前提の大規模変更

## 安全ルール

安全策や追加条件を入れる場合は、必ず説明してください。

説明する内容は以下です。

・何を追加したか
・なぜ追加したか
・悪い影響はあるか
・入れない場合と何が違うか

ユーザーが明示していない条件を、黙って追加しないでください。

## 今日のコード1行解説ルール

毎回、コードの中から1行を選び、小学生にも分かるように説明してください。

例：

```python
if game_over:
```

説明：

ゲームが終わった状態かどうかを確認する命令です。

「今はゲーム中？それともゲームオーバー？」をチェックしています。

## フォルダ構成方針

今後、新しいゲームを作る場合は、同じリポジトリ内でゲームごとにフォルダを分けます。

例：

```text
aiworkslab-game-project
├ README.md
├ CODEX.md
├ .gitignore
├ games
│  ├ game01_dodge
│  │  ├ game01_dodge.py
│  │  └ README.md
│  ├ game02_stardust
│  │  ├ README.md
│  │  └ main.py
│  └ game03_dark_survival
│     ├ README.md
│     └ main.py
├ docs
└ assets
```

ただし、ユーザーが理解できなくなるような大規模なフォルダ変更は、事前に説明してから行ってください。
